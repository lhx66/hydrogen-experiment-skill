## 一、系统架构

### 1.1 核心组件

```
自动化实验系统
├── 命令行工具集 (CLI Tools)
│   ├── mfc_cli.exe          - MFC质量流量控制器命令行工具
│   ├── powermeter_cli.exe   - 功率计命令行工具
│   └── fbg_cli.exe          - FBG解调仪命令行工具
├── 数据分析模块 (Analysis)
│   ├── analyze_sensor_response.py  - 传感器响应分析
│   └── analyze_fbg_response.py    - FBG数据分析
├── 自动化Skill (Skill)
│   └── hydrogen_experiment_skill   - 实验自动化skill
└── 实验数据目录
    └── experiments/
        ├── 2026-06-17_sensor_A_h2_test/
        └── ...
```

### 1.2 工作流程

```
用户下达实验指令
    ↓
Agent解析参数 → 生成实验方案 → 确认
    ↓
执行实验循环:
  1. 打开MFC2（载气） → 等待稳定
  2. 启动数据记录（功率计或FBG）
  3. 打开MFC1（氢气） → 等待指定时间
  4. 关闭MFC1 → 等待恢复
  5. 停止数据记录
  6. 数据分析 → 评估结果
    ↓
循环结束 → 关闭所有设备 → 生成实验报告
```

## 二、命令行工具规格

### 2.1 MFC控制工具 (mfc_cli.exe)

#### 功能要求

1. **设备通信**
   - 支持MODBUS RTU协议
   - 双MFC通道：
     - MFC1（地址1）：氢气控制，单位sccm
     - MFC2（地址2）：载气控制，单位slm
   - 串口参数可配置

2. **安全机制**
   - 实时监测MFC2流量
   - 当MFC2返回值 < 0.1 slm时，立即关闭MFC1
   - 防止氢气浓度过高

3. **命令行接口**

```bash
# 连接设备
mfc_cli.exe connect --port COM3

# 设置单个MFC流量
mfc_cli.exe set --channel 2 --flow 1.0   # 设置MFC2为1.0 slm载气
mfc_cli.exe set --channel 1 --flow 30    # 设置MFC1为30 sccm，对应3%氢气

# 读取当前流量
mfc_cli.exe read --channel 1

# 执行实验流程（核心命令）
mfc_cli.exe run-sequence \
    --mfc2-flow 1.0 \              # 载气流量 (slm)
    --mfc1-flow 30 \               # 氢气流量 (sccm)，对应3%氢气
    --mfc1-duration 40 \           # 通氢气时间 (秒)
    --loop-count 10 \              # 循环次数
    --loop-interval 60 \           # 循环间隔 (秒)
    --pre-mfc2-time 30             # 首次打开MFC2前的等待时间 (秒)

# 关闭所有MFC
mfc_cli.exe all-close

# 断开连接
mfc_cli.exe disconnect
```

#### run-sequence 命令执行流程

```
1. 打开MFC2到设定流量，等待 pre_mfc2_time 秒
2. 开始循环（loop_count次）：
   a. 打开MFC1到设定流量
   b. 等待 mfc1_duration 秒
   c. 关闭MFC1
   d. 等待 loop_interval 秒（可继续数据采集）
3. 循环结束后，先关闭MFC1，再关闭MFC2
4. 输出执行日志
```

### 2.3 FBG解调仪工具 (fbg_cli.exe)

#### 功能要求

1. **设备通信**
   - TCP连接到解调仪
   - 接收8通道波长数据（100Hz）

2. **命令行接口**

```bash
# 连接并启动数据采集；connect只用于连通性检查
fbg_cli.exe start \
    --ip 192.168.1.1 \
    --duration 600 \
    --filename sensor_A_H2-3percent_MFC1-30sccm_MFC2-1slm_H2time-40s_Record-600s_FBG-ch1_cycle01 \
    --channel 1 \              # 选择通道（1-8）
    --moving-average 1        # 滑动平均窗口
```

CSV输出格式：


## 三、数据分析模块

### 3.1 传感器响应分析 (analyze_sensor_response.py)

#### 功能

1. **检测响应**
   - 基线稳定性判断
   - 响应起始点检测（滑动窗口 + 3σ准则）
   - 响应幅度计算

2. **响应时间 t90**
   - 定义：从响应起始点到达到90%响应幅度的时间
   - 自动检测基线值和稳态值

3. **恢复时间**
   - 定义：从关闭氢气到恢复到基线10%范围内的时间

4. **滑动窗口检测算法**

```python
def detect_response_start(data, window_size=30, n_sigma=3, consecutive_n=5):
    """
    检测响应起始点

    算法：
    1. 第一个滑动窗口：计算基线均值和标准差
    2. 第二个滑动窗口：检测超出3σ的点
    3. 连续出现N个超出点时，第一个点为起始点

    参数：
        data: 数据数组
        window_size: 滑动窗口大小
        n_sigma: sigma阈值倍数
        consecutive_n: 连续超出点数

    返回：
        start_index: 起始点索引
        baseline: 基线平均值
    """
```

5. **输出**

```json
{
    "has_response": true,
    "response_start_time": 12.34,
    "response_amplitude": 0.0234,
    "baseline_value": 1547.1234,
    "steady_value": 1547.1468,
    "t90": 8.5,
    "recovery_time": 15.2,
    "signal_to_noise": 45.6,
    "concentration_estimated": "4%"
}
```

## 四、自动化Skill设计

### 4.1 Skill功能

```python
# hydrogen_experiment_skill.py

class HydrogenExperimentSkill:
    """光纤氢气传感器实验自动化skill"""

    def run_experiment(self,
                      sensor_name: str,
                      concentration: str,
                      h2_time: int,
                      cycles: int,
                      instrument: str,
                      total_duration: int):
        """
        运行一次完整实验

        参数：
            sensor_name: 传感器名称（如 "FBG_Sensor_01"）
            concentration: 氢气浓度（如 "4%"）
            h2_time: 每次通氢气时间（秒）

        """
```

### 4.2 实验执行流程

```
1. 参数验证
2. 生成实验ID和数据目录
3. 启动MFC run-sequence（后台）
4. 启动数据记录（后台）
5. 监控进度
6. 循环完成后停止所有进程
7. 对每次循环进行数据分析
8. 生成实验报告
9. 根据分析结果提供建议
```

## 七、错误处理与安全

### 7.1 安全机制

1. **MFC安全**
   - 载气流量监测阈值：0.1 slm
   - 超时自动关闭
   - 异常断开保护

2. **数据安全**
   - 定期flush数据
   - 异常时保存已采集数据
   - 文件命名防止覆盖

### 7.2 错误处理

1. 设备通信失败 → 重试 → 报警
2. 数据异常 → 记录 → 继续
3. 用户中断 → 优雅退出 → 保存数据
